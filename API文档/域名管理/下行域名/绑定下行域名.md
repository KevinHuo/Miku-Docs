# 绑定下行域名
## 接口描述
为指定直播空间绑定下行域名（播放域名）

## 接口信息
- **请求方法**: POST
- **请求路径**: /
- **请求参数**:
  - **domain** (string) - 域名配置，必需

## 请求参数
### Header 参数
|   参数名	|   位置	 |   类型	   |必填	 |示例值	|描述|
|:--------- |:--------|:-----------|:-----|:---------|:---|
|   Host     |	Header|	string   |	是   | `<bucket>`.mls.cn-east-1.qiniumiku.com|	请求的服务域名|
|   Authorization | Header | string | 是 | `<QiniuToken>` |  管理凭证 QiniuToken |

### Body 参数 (application/json)
示例：
```json
{
    "domain":"www.example.com",
    "type":"liveRtmp"
}
```
### 参数
| 参数名称 | 类型 | 说明 | 是否必需 |
|:------|:------|:------|:----------|
| domain | string | 要绑定的域名 | 必需 |
| type | string | 域名类型 (liveRtmp : rtmp 播放域名; liveHls : hls 播放域名; liveDash : dash 播放域名; liveFlv : flv 播放域名; whep : whep 播放域名; live : 所有的播放协议共用一个域名; liveSrt : srt 播放域名) | 可选 |

## 访问域名
```go
<bucket>.mls.cn-east-1.qiniumiku.com
```

## 认证授权
```go
Authorization: <QiniuToken>
```
- **QiniuToken 详细描述请查阅**: [这里](/docs/public/请求鉴权)

## 请求示例
```http
POST /?domain HTTP/1.1
Host: <bucket>.mls.cn-east-1.qiniumiku.com
Authorization: <QiniuToken>
Content-Type: application/json

{
    "domain":"www.example.com",
    "type":"liveRtmp"
}
```

### 响应状态码
- **请查阅**: [这里](/docs/public/错误返回)


### 响应参数
| 参数名称 | 类型 | 说明 | 是否必需 |
|:------|:------|:------|:---------|
| domain | string | 域名 | 必需 |
| cname | string | cname | 必需 |
| creationDate | string | 创建时间 | 必需 |
| lastModified | string | 修改时间 | 必需 |
| connectId | string | 连接 id | 可选 |
| type | string | 域名类型 | 必需 |
