# 绑定上行域名
## 接口描述
为指定直播空间绑定上行域名（推流域名）

## 接口信息
- **请求方法**: POST
- **请求路径**: /
- **请求参数**:
  - **pushDomain** (string) - 域名配置，必需

## 请求参数
### Header 参数
|   参数名	|   位置	 |   类型	   |必填	 |示例值	|描述|
|:--------- |:--------|:-----------|:-----|:---------|:---|
|   Host     |	Header|	string   |	是   | `<bucket>`.mls.cn-east-1.qiniumiku.com|	请求的服务域名|
|   Authorization | Header | string | 是 | `<QiniuToken>` |  管理凭证 QiniuToken |

### Body 参数 (application/json)
示例：
```json
{
    "domain":"www.example.com",
    "type":"pushRtmp"
}
```
### 参数
| 参数名称 | 类型 | 说明 | 是否必需 |
|:------|:------|:------|:----------|
| domain | string | 域名 | 必需 |
| type | string | 域名类型（pushRtmp:推流域名;whip:whip推流域名;pushSrt推流域名;）| 必须 |

## 访问域名
```go
<bucket>.mls.cn-east-1.qiniumiku.com
```

## 认证授权
```go
Authorization: <QiniuToken>
```
- **QiniuToken 详细描述请查阅**: [这里](/docs/public/请求鉴权)

## 请求示例
```http
POST /?pushDomain HTTP/1.1
Host: <bucket>.mls.cn-east-1.qiniumiku.com
Authorization: <QiniuToken>
Content-Type: application/json

{
    "domain":"www.example.com",
    "type":"pushRtmp"
}
```

### 响应状态码
- **请查阅**: [这里](/docs/public/错误返回)

```json
{
```

### 响应参数
| 参数名称 | 类型 | 说明 | 是否必需 |
|:------|:------|:------|:----------|
| domain | string | 域名 | 必需 |
| cname | string | cname | 必需 |
| creationDate | string | 创建时间 | 必需 |
| lastModified | string | 修改时间 | 必需 |
| connectId | object | 连接 id | 必需 |
| type | string | 域名类型 | 必需 |
